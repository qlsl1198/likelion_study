2주차 예상 질문 · React 기초

React는 프레임워크인가요, 라이브러리인가요?
공식적으로는 UI를 만드는 데 집중한 자바스크립트 라이브러리로 소개됩니다. 앱 전체 구조를 모두 정하는 “풀 프레임워크”라기보다, 화면(컴포넌트) 쪽에 가깝게 쓰는 경우가 많아요.
왜 HTML의 class 대신 className을 쓰나요?
JSX는 자바스크립트 안에서 쓰이고, JS에서 class는 예약어라서 충돌을 피하려고 className을 씁니다.
가상 DOM(Virtual DOM)이 뭔가요?
브라우저가 가진 실제 DOM을 매번 통째로 바꾸지 않고, React가 메모리에 둔 가벼운 복사본끼리 비교(diff)해서 바뀐 부분만 실제 화면에 반영하려는 방식으로 이해하면 됩니다.
Props와 State는 어떻게 구분하나요?
Props는 부모가 자식에게 넘기는 값(읽기 위주). State는 그 컴포넌트 안에서 바뀌는 데이터입니다. “버튼 눌러서 이 값이 바뀌나?” → 바뀌면 State, 부모가 정해준 고정 값이면 Props 쪽을 먼저 생각해 보세요.
useState가 [값, 함수] 두 개를 주는 이유는요?
첫 번째는 현재 state 값, 두 번째는 그 값을 바꿀 때만 써야 하는 setter입니다. state 변수에 직접 대입하면 React가 변경을 제대로 반영하지 못할 수 있어요.
setCount(count + 1)과 setCount(prev => prev + 1) 차이는요?
연속으로 여러 번 업데이트할 때는 이전 값에 기반하는 함수형 업데이트(prev => prev + 1)이 안전합니다. 한 번에 한 번만 올리면 둘 다 비슷하게 동작하는 경우가 많아요.
return 안에 왜 태그를 하나의 부모로 감싸야 하나요?
JSX는 하나의 루트를 반환해야 하는 규칙이 있습니다. 불필요한 div가 싫으면 Fragment(<>...</>)로 감쌀 수 있어요.
컴포넌트 이름은 왜 대문자로 시작하나요?
소문자로 시작하면 HTML 태그로 해석될 수 있어요. React 컴포넌트는 대문자로 시작하는 게 규칙입니다.
onClick에 왜 문자열이 아니라 함수를 넣나요?
React에서는 중괄호 안에 자바스크립트 함수를 넘깁니다. DOM 옛 방식의 문자열 속성과 다릅니다.
폼에서 e.preventDefault()는 왜 쓰나요?
기본 제출 동작(페이지 새로고침 등)을 막고, 자바스크립트로만 처리하려고 씁니다.
create-react-app 말고 Vite 등 다른 도구는요?
실무에서는 Vite 등을 많이 쓰기도 합니다. 강의 슬라이드는 입문용으로 CRA를 안내한 것이니, 개념은 같고 도구만 다르다고 이해하시면 됩니다.
