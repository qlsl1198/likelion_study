1. React Hooks 소개 (30분)

1.1 Hooks란?

  • 정의: 함수형 컴포넌트에서 State와 생명주기 기능을 사용할 수 있게 해주는 함수
  • 규칙:
    • 최상위 레벨에서만 호출
    • 조건문, 반복문, 중첩 함수에서 호출 금지
    • React 함수 컴포넌트 또는 커스텀 Hook에서만 호출
  • 이유: React가 Hook 호출 순서로 상태를 추적. 순서가 바뀌면 상태가 꼬임

1.2 주요 Hooks

  • useState: State 관리
  • useEffect: 사이드 이펙트 처리
  • useContext: Context 사용
  • useRef: DOM 참조 또는 값 저장
  • useMemo: 값 메모이제이션
  • useCallback: 함수 메모이제이션

2. useEffect (90분)

2.1 useEffect란?

  • 정의: 컴포넌트가 렌더링된 후 실행되는 Hook
  • 용도: 데이터 가져오기, 구독 설정, DOM 수동 조작, 타이머 설정
  • 의존성 배열:
    • 없음: 매 렌더링마다 실행
    • []: 마운트 시 1회만
    • [a, b]: a나 b가 바뀔 때만 실행

2.2 기본 사용법

import { useEffect } from 'react';

function Component() {
    useEffect(() => {
        console.log('Component mounted');
        return () => {
            console.log('Component unmounted');
        };
    });
    return <div>Content</div>;
}

2.3 의존성 배열

  • 없음: 매 렌더링마다 실행
  • []: 마운트 시 한 번만 (API 호출, 구독 설정 등)
  • [a, b]: a 또는 b가 변경될 때마다 실행

2.3 의존성 배열 - 예제

// 매 렌더링마다 실행
useEffect(() => { console.log('Every render'); });

// 마운트 시 한 번만 실행
useEffect(() => { console.log('Only on mount'); }, []);

// 특정 값 변경 시 실행
useEffect(() => { console.log(count); }, [count]);

2.4 데이터 가져오기

function UserProfile({ userId }) {
    const [user, setUser] = useState(null);
    const [loading, setLoading] = useState(true);
    
    useEffect(() => {
        async function fetchUser() {
            setLoading(true);
            const response = await fetch(`/api/users/${userId}`);
            const data = await response.json();
            setUser(data);
            setLoading(false);
        }
        fetchUser();
    }, [userId]);
    
    if (loading) return <div>Loading...</div>;
    return <div>{user.name}</div>;
}

2.5 클린업 함수

function Timer() {
    const [seconds, setSeconds] = useState(0);
    
    useEffect(() => {
        const interval = setInterval(() => {
            setSeconds(prev => prev + 1);
        }, 1000);
        return () => clearInterval(interval);
    }, []);
    
    return <div>Seconds: {seconds}</div>;
}

3. useContext (60분)

3.1 Context API란?

  • 정의: Props drilling을 피하기 위한 전역 상태 관리 방법
  • Props drilling: App → A → B → C처럼 여러 단계를 거쳐 props를 내려보내는 방식. 중간 컴포넌트는 해당 props를 사용하지도 않는데 전달만 하는 비효율 발생
  • Context: Provider로 감싼 영역 안의 모든 컴포넌트가 직통으로 데이터 접근

3.2 Context 생성 및 사용

const ThemeContext = createContext('light');

function App() {
    const [theme, setTheme] = useState('light');
    return (
        <ThemeContext.Provider value={{ theme, setTheme }}>
            <Header /><Content />
        </ThemeContext.Provider>
    );
}

function Header() {
    const { theme, setTheme } = useContext(ThemeContext);
    return <button onClick={() => setTheme(...)}>Toggle</button>;
}

4.1 React Router란?

  • 정의: React 애플리케이션에서 라우팅을 처리하는 라이브러리
  • 설치: npm install react-router-dom

4.2 기본 설정

import { BrowserRouter, Routes, Route } from 'react-router-dom';

function App() {
    return (
        <BrowserRouter>
            <Routes>
                <Route path="/" element={<Home />} />
                <Route path="/about" element={<About />} />
            </Routes>
        </BrowserRouter>
    );
}

4.3 Link와 NavLink

<Link to="/">Home</Link>

5. 폼 처리 및 검증 (90분)

5.1 제어 컴포넌트

function LoginForm() {
    const [email, setEmail] = useState('');
    const [password, setPassword] = useState('');
    
    return (
        <form onSubmit={(e) => { e.preventDefault(); ... }}>
            <input value={email} onChange={(e) => setEmail(e.target.value)} />
            <input value={password} onChange={(e) => setPassword(e.target.value)} />
            <button type="submit">Login</button>
        </form>
    );
}

5.2 폼 검증

  • validate 함수로 이메일 형식, 비밀번호 길이 등 검증
  • errors 상태로 에러 메시지 저장
  • handleSubmit에서 검증 후 통과 시에만 제출
