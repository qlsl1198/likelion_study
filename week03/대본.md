# 3주차 강의 대본 — React 심화 (Hooks, Router, 폼)

**예상 시간**: 약 3~3.5시간  
**준비물**: 2주차 React 프로젝트, `react-router-dom` 설치 가능한 네트워크

---

## 오프닝 (5~10분)

> "지난주에 컴포넌트랑 State까지 했죠. 오늘은 **Hooks**로 **부가 작업**을 처리하고, **여러 페이지**를 쓰는 **React Router**, 그리고 **폼**을 다룹니다.  
> 한 줄로 말하면 **'실제 앱처럼 보이게 만드는 주차'**예요."

- 수업 전 안내: "프로젝트 폴더에서 `npm install react-router-dom` 해 주세요"

---

## 1. Hooks 개요 (약 20~30분)

> "Hooks는 **함수형 컴포넌트**에서만 쓰는 규칙이에요. **최상위에서만** 호출한다는 규칙이 왜 있냐면, React가 **호출 순서**로 상태를 기억하기 때문이에요."

- 조건문 안에 Hook 넣으면 안 되는 이유: 짧게 한 줄로.

---

## 2. useEffect (약 60~70분)

> "렌더링이 끝난 **뒤**에 해야 할 일—API 호출, 타이머, 구독—은 **useEffect**로 빼요."

- 의존성 배열:
  - `[]`: "마운트될 때 한 번"
  - `[a]`: "a가 바뀔 때마다"
  - 없음: "거의 매 렌더링마다—**의도적으로 쓸 일이 거의 없어요**"
- 클린업: "타이머 정리, 구독 해제—**메모리 누수** 막기"

**데모**: `useEffect`로 `console.log` 또는 간단한 fetch

**멘트**: "초보자가 제일 많이 하는 실수가 **의존성 배열을 빼먹는 것**이에요."

---

## 3. useContext (약 30~40분)

> "Props를 여러 층으로만 넘기면 **Props drilling**이라고 불러요. 전역으로 쓸 값은 **Context**로 빼는 패턴이 있어요."

- "남용은 금물—**정말 여러 깊이에서 쓰는 값**일 때만"

---

## 4. React Router (약 50~60분)

> "SPA는 페이지를 **새로고침 없이** 바꿔요. Router가 **URL과 화면**을 연결해 줘요."

- `BrowserRouter`, `Routes`, `Route`, `Link`
- `useParams`, `useNavigate`: "URL 파라미터, 프로그램적으로 이동"

**데모**: `/`와 `/about` 두 페이지만

---

## 5. 폼 & 검증 (약 40~50분)

> "제어 컴포넌트—**input의 value를 state로 묶는다**는 뜻이에요."

- 간단한 검증: "제출할 때만 체크해도 되고, 입력할 때마다 해도 돼요"

---

## 6. 실습·마무리 (20분)

> "Todo에 라우트 나누기, Context로 테마 하나 넣기 같은 걸 과제로 연결하세요."
