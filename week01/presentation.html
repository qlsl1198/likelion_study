1. 개발 환경 구축 (60분)

1.1 Node.js 설치

Node.js란?

💡 한 줄 요약: 원래 브라우저 안에서만 돌아가던 JavaScript를, 컴퓨터 어디서나 실행하게 해주는 프로그램

  • 런타임: JS를 "실행해주는 환경". 브라우저 없이도 JS 코드가 돌아감
  • npm: 다른 사람이 만든 코드(라이브러리)를 받아 쓰는 도구. 앱스토어처럼 "패키지 설치"하는 느낌

Node.js 설치 방법 - 따라하기 (1/2)

1단계: nodejs.org/ko 접속

2단계: LTS 버전 (초록색 버튼) 다운로드 (Windows: .msi / Mac: .pkg)

Node.js 설치 방법 - 따라하기 (2/2)

3단계: 설치 실행 → Windows: Next 반복 / Mac: 계속 → 동의 → 설치

4단계: 터미널 열기 (Win+R→cmd / Mac: Cmd+Space→터미널) → 버전 확인:

node --version
npm --version

✅ v18 이상 나오면 성공!

1.2 Java JDK 설치

Java란?

  • 객체지향 프로그래밍 언어 → 코드를 객체 단위로 구조화
  • Spring Boot는 Java 기반 → 백엔드 개발에 필수
  • JDK vs JRE: JDK = 개발용 (컴파일러 포함), JRE = 실행용만

Java JDK 설치 - 따라하기 (1/2)

1단계: adoptium.net/ko 접속

2단계: JDK 17 또는 21 LTS → Windows: .msi / Mac: .pkg 선택

3단계: 설치 실행

  • Windows: 기본 경로 그대로
  • Mac: 계속 → 동의 → 설치

Java JDK 설치 - 따라하기 (2/2)

4단계: 터미널에서 확인 (Mac: Cmd+Space → "터미널"):

java -version
javac -version

✅ 버전이 나오면 성공!

Java 환경 변수 (Windows)

java 명령이 안 되면

  1. Windows 검색창에 "환경 변수" 입력
  2. "시스템 환경 변수 편집" → "환경 변수" 클릭
  3. JAVA_HOME: 새로 만들기 → C:\Program Files\Eclipse Adoptium\jdk-17.x.x
  4. Path 편집 → %JAVA_HOME%\bin 추가

Java 환경 변수 (Mac) (1/2)

macOS는 보통 자동 설정. 안 되면:

# 터미널에서 (Cmd+Space → 터미널)
nano ~/.zshrc

아래 두 줄 추가 후 저장 (Ctrl+O → 엔터 → Ctrl+X):

export JAVA_HOME=$(/usr/libexec/java_home)
export PATH=$JAVA_HOME/bin:$PATH

Java 환경 변수 (Mac) (2/2)

저장 후 터미널 다시 열기

또는 source ~/.zshrc 실행

2. 웹 개발 기초 개념 (90분)

2.1 웹의 동작 원리

클라이언트-서버 모델

💡 쉬운 비유: 레스토랑에서 손님(클라이언트)=브라우저, 점원/주방(서버)=웹서버

  • 손님이 "이 음식 주세요" → 요청(Request)
  • 점원이 음식 전달 → 응답(Response)
  • 웹: URL 입력 → 서버가 HTML/데이터 전달 → 화면에 표시

2.1 웹의 동작 원리 (계속)

브라우저(클라이언트) → HTTP 요청 → 서버
브라우저(클라이언트) ← HTTP 응답 ← 서버

HTTP 프로토콜

Stateless(무상태): 서버는 이전 요청 정보를 저장하지 않음. 각 요청을 독립적으로 처리

  • GET: 데이터 조회 (URL 파라미터, 캐싱 가능)
  • POST: 데이터 생성 (요청 본문에 데이터 포함)
  • PUT: 리소스 전체 수정
  • DELETE: 리소스 삭제

3. Git/GitHub 기초 (60분)

3.1 Git 기본 명령어 (1/2)

# 저장소 초기화
git init

# 파일 추가
git add .

# 커밋
git commit -m "Initial commit"

3.1 Git 기본 명령어 (2/2)

# 상태 확인
git status

# 변경 이력 확인
git log

3.1 (퀵 정리) add → commit → push

  • git add: 변경을 스테이징에 올림
  • git commit: 스테이징을 스냅샷으로 기록
  • git push: 커밋을 GitHub(원격)에 전송
# (일반 흐름)
git status
git add .
git commit -m "..."
git push

최초 1회만: git push -u origin main

3.2 GitHub 레포지토리 생성 - 따라하기

  • GitHub → New repository
  • Repository name: likelion-study-week01 (예시)
  • Public/Private 선택
  • README 추가: 선택하지 않음
  • 저장소 생성 후 저장소 URL 복사

3.2 GitHub 연결 - 따라하기

# (선택) 로컬 브랜치가 master면 main으로 맞추기
git branch -M main

# 원격 저장소 연결 (URL은 GitHub에서 복사)
git remote add origin https://github.com/username/repo.git

# 연결 확인
git remote -v

3.2 GitHub 첫 푸시 - 따라하기

# 최초 푸시(업스트림 설정)
git push -u origin main

3.2 (참고) Pull / Clone

# 원격 변경사항 가져오기 (pull)
git pull origin main

# 저장소 복제 (clone)
git clone https://github.com/username/repo.git

3.3 브랜치 관리

브랜치: main에서 분기된 독립 작업 공간. 기능 개발·버그 수정을 분리 후 병합

3.3 브랜치 관리 (명령어)

# 브랜치 생성 및 이동
git checkout -b feature/new-feature

# 브랜치 목록
git branch

# 브랜치 병합
git merge feature/new-feature

4. 프로젝트 구조 이해 (30분)

4.1 프론트엔드 프로젝트 구조

  • components/: 재사용 UI 조각 (버튼, 카드 등)
  • pages/: 화면 단위 (홈, 로그인 등)
  • utils/: 공통 유틸 함수

4.1 프론트엔드 프로젝트 구조 (폴더 트리)

frontend/
├── public/index.html
├── src/
│   ├── components/
│   ├── pages/
│   ├── utils/
│   ├── App.js
│   └── index.js
├── package.json
└── README.md

4.2 백엔드 프로젝트 구조

  • controller/: HTTP 요청 처리
  • service/: 비즈니스 로직
  • repository/: DB 접근 (CRUD)
  • entity/: DB 테이블 매핑 클래스

4.2 백엔드 프로젝트 구조 (폴더 트리)

backend/
├── src/main/java/.../controller/
│   service/, repository/, entity/
├── src/main/resources/
│   application.properties
├── src/test/
├── pom.xml (또는 build.gradle)
└── README.md
