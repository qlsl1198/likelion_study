# 1주차 강의 대본 — 개발 환경 & 웹 기초

**예상 시간**: 약 3시간 (휴식·실습 포함 시 3.5~4시간)  
**준비물**: 슬라이드, 데모용 노트북(터미널·브라우저), 수강생은 노트북 필수

---

## 오프닝 (5~10분)

> "안녕하세요. 오늘부터 **풀스택**이라는 말을 자주 쓸 거예요. 한마디로 **화면도 만들고(프론트), 서버·DB도 다루는(백엔드)** 흐름을 같이 가는 거라고 생각하시면 됩니다.  
> 이번 주는 **도구를 깔고**, **웹이 어떻게 돌아가는지**, **Git으로 코드를 어떻게 관리하는지**까지 잡습니다. 오늘 끝나면 '내 컴퓨터에서 React랑 Spring 예제를 돌려볼 수 있는 상태'가 목표예요."

- 학습 목표 슬라이드에서 네 줄만 천천히 읽어 주기.
- 비전공자에게: "터미널 = 검은 창에 글자로 명령 넣는 곳" 한 번 더 강조.

---

## 1. 개발 환경 구축 (약 60~70분)

### Node.js (10~15분)

> "자바스크립트는 원래 브라우저 안에서만 돌았어요. **Node.js**는 그 JS를 **컴퓨터 어디서나** 돌리게 해 주는 실행 환경이에요. 우리는 나중에 React를 쓰기 위해 `npm`으로 패키지를 깔 거고, 그게 Node랑 같이 옵니다."

- LTS 버전 다운로드 이유: "최신 기능보다 **안정적으로 강의 따라오기** 좋은 버전"
- **데모**: 터미널 열기 → `node -v`, `npm -v` 한 번에 보여주기.
- 막히는 사람: 터미널 완전히 닫았다가 다시 열기, 재부팅 안내.

### Java JDK (10~15분)

> "백엔드는 **Spring Boot**를 쓸 건데, Spring은 **Java** 위에서 돌아가요. 그래서 JDK가 필요합니다. JDK는 **개발할 때** 필요한 풀세트, JRE는 실행만 할 때 쓰는 말이에요."

- Adoptium / Oracle 중 강의에서 쓰는 것 하나로 통일해 말하기.
- **데모**: `java -version`, `javac -version`
- Windows 환경 변수: "명령이 안 되면 이 경로를 컴퓨터가 모르는 거예요"라고 직관적으로.

### VS Code · IntelliJ · Git (20~25분)

> "앞으로 코드는 **VS Code**에서 주로 보고, Java 백엔드는 **IntelliJ**가 편해요. 둘 다 설치해 두는 걸 권장합니다."

- VS Code 확장: Prettier, ESLint, React 스니펫 — "자동완성·포맷팅 때문에 시간이 줄어요"
- Git: "혼자만 쓰는 백업이 아니라 **협업과 이력**의 기본이에요"
- **데모**: `git --version`, `git config user.name` / `user.email` 설정

**멘트**: "여기까지 설치가 안 되면 뒤 주차가 전부 막혀요. **오늘 안에** 반드시 끝까지 확인해 주세요."

## 2. 웹 개발 기초 (약 80~90분)

### 클라이언트–서버 & HTTP (20분)

> "브라우저가 **요청**을 보내고, 서버가 **응답**을 주는 구조예요. 레스토랑에 비유하면 손님이 브라우저, 주방이 서버라고 슬라이드에 나와 있죠."

- GET/POST/PUT/DELETE: "일단 **GET은 가져오기**, **POST는 새로 만들기** 정도만 기억해도 됩니다"

### HTML · CSS · JS (각 15~20분)

> "HTML은 **뼈대**, CSS는 **스타일**, JS는 **움직임**이에요. 세 가지가 한 파일에 있을 수도 있고 나눌 수도 있어요."

- 짧은 예제 코드 슬라이드 위에서 한 줄씩 짚어 주기.
- "비전공자는 **태그 이름**이랑 **역할**만 먼저 익히면 됩니다"

---

## 3. Git / GitHub (약 50~60분)

> "init은 저장소 만들기, add는 무대에 올리기, commit은 **스냅샷 찍기**예요."

- `branch`, `merge`: "기능별로 나눠 작업했다가 합친다"만 설명해도 충분.
- GitHub: "원격에 올리고, 팀원과 공유하는 곳"

---

## 4. 프로젝트 구조 & 실습 (약 30~40분)

> "프론트는 `components`, `pages`처럼 **역할별 폴더**로 나누는 게 일반적이고, 백엔드는 `controller → service → repository`로 요청이 흘러가요."

- CRA / Spring Initializr는 **개념만** 짚고, 시간 있으면 짧게 데모.
- "오늘은 **구조를 머릿속에 그림** 그리는 게 목표"

---

## 5. 마무리 & 과제 안내 (10분)

> "오늘은 **도구 설치 + 웹·Git 큰 그림**이에요. 막히는 부분은 채널에 올려 주시고, 과제는 **개인 저장소** 하나 만들고 HTML 페이지 하나 올려 보는 것부터요."

- `exercises.md` 또는 조직 과제 저장소 링크 안내.
- 다음 주 예고: "2주차부터 **React** 들어갑니다. 공식 문서 첫 페이지만 읽어 오시면 좋아요."
