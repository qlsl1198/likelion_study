## 실습 2: 회원가입 및 로그인 API (50분)

### 목표
회원가입과 로그인 API를 구현합니다.

### 문제
다음 기능을 구현하세요:

1. **회원가입 API**
   - POST `/api/auth/signup`
   - 사용자명, 비밀번호, 이메일 입력
   - 비밀번호 암호화 저장
   - 중복 사용자명 체크

2. **로그인 API**
   - POST `/api/auth/login`
   - 사용자명과 비밀번호로 인증
   - JWT 토큰 발급 및 반환

3. **에러 처리**
   - 잘못된 자격증명 처리
   - 중복 사용자명 처리

### 제출물
- AuthController.java
- AuthService.java
- DTO 클래스들
- Postman 테스트 결과

## 실습 3: Spring Security 설정 (40분)

### 목표
Spring Security를 설정하여 인증을 적용합니다.

### 문제
다음 설정을 구현하세요:

1. **SecurityConfig**
   - 인증이 필요한 경로 설정
   - 공개 경로 설정 (`/api/auth/**`)
   - JWT 필터 추가

2. **JWT 필터**
   - 요청 헤더에서 토큰 추출
   - 토큰 검증
   - 인증 정보 설정

3. **테스트**
   - 인증 없이 접근 시 401 에러
   - 토큰과 함께 접근 시 정상 작동

### 제출물
- SecurityConfig.java
- JwtAuthenticationFilter.java
- 테스트 결과

## 실습 4: React 인증 구현 (50분)

### 목표
React에서 인증 상태를 관리하고 보호된 라우트를 구현합니다.

### 문제
다음 기능을 구현하세요:

1. **AuthContext**
   - 사용자 상태 관리
   - login, signup, logout 함수
   - 토큰 저장 및 관리

2. **Login/Signup 컴포넌트**
   - 로그인 폼
   - 회원가입 폼
   - 에러 처리

3. **ProtectedRoute**
   - 인증되지 않은 사용자 리다이렉트
   - 인증된 사용자만 접근 가능

4. **Axios 인터셉터**
   - 요청에 토큰 자동 추가
   - 401 에러 시 로그아웃 처리

### 제출물
- AuthContext.js
- Login.js, Signup.js
- ProtectedRoute.js
- 실행 결과 스크린샷

## 실습 5: 사용자별 Todo 관리 (40분)

### 목표
인증된 사용자만 자신의 Todo를 관리할 수 있도록 구현합니다.

### 문제
다음 기능을 구현하세요:

1. **백엔드 수정**
   - Todo 엔티티에 User 관계 추가
   - 사용자별 Todo 조회
   - 다른 사용자의 Todo 접근 방지

2. **프론트엔드 수정**
   - 로그인한 사용자의 Todo만 표시
   - 새 Todo 생성 시 현재 사용자 연결

3. **테스트**
   - 로그인 후 Todo 생성/조회
   - 로그아웃 후 접근 불가 확인

### 제출물
- 수정된 백엔드 코드
- 수정된 프론트엔드 코드
- 테스트 결과
