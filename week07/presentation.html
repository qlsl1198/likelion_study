1. JWT 인증 소개

1.1 JWT란?

  • 정의: JSON Web Token - 인증 정보를 안전하게 전송하기 위한 토큰
  • Stateless: 서버에 세션 저장 불필요 → 서버 확장에 유리

1.1 JWT 구조 (계속)

  • Header: 알고리즘, 토큰 타입
  • Payload: 사용자 ID, 만료 시간 등 (암호화 X, Base64 인코딩)
  • Signature: 서버 비밀키로 서명 → 위변조 검증

형식: Header.Payload.Signature

1.2 JWT 동작 방식

1. 사용자 로그인
2. 서버에서 JWT 생성 및 반환
3. 클라이언트가 JWT 저장 (localStorage 등)
4. 이후 요청에 JWT 포함 (Authorization 헤더)
5. 서버에서 JWT 검증

JWT·Spring Security 의존성 추가 - 따라하기 (2/2)

추가할 의존성:

  • spring-boot-starter-security
  • jjwt-api, jjwt-impl, jjwt-jackson

2.1 Spring Security 의존성 (pom.xml)

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>
<dependency>
    <groupId>io.jsonwebtoken</groupId>
    <artifactId>jjwt-api</artifactId>
    <version>0.11.5</version>
</dependency>

2.2 Security 설정

@Configuration
@EnableWebSecurity
public class SecurityConfig {
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) {
        http.csrf().disable()
            .sessionManagement().sessionCreationPolicy(STATELESS)
            .authorizeHttpRequests()
                .requestMatchers("/api/auth/**").permitAll()
                .anyRequest().authenticated()
            .addFilterBefore(new JwtAuthenticationFilter(...), ...);
        return http.build();
    }
}

3. React에서 인증 처리

  • 로그인 시 JWT를 localStorage에 저장
  • Axios 인터셉터로 요청 시 Authorization 헤더에 토큰 추가
