# 4주차 강의 대본 — Spring Boot 기초

**예상 시간**: 약 3시간  
**준비물**: JDK, IntelliJ, Postman(또는 설치 중), 슬라이드의 Spring Initializr 설정과 동일

---

## 오프닝 (5~10분)

> "지금까지는 브라우저 쪽, **React**를 봤죠. 이제 **서버**를 만듭니다. **Spring Boot**는 Java로 **API 서버**를 빠르게 띄우게 해 주는 프레임워크예요.  
> 오늘 목표는 **프로젝트 생성**, **실행**, **MVC 구조**, **REST API 한 개**를 직접 쳐 보는 거예요."

- MVC 레스토랑 비유: "손님 주문 = Controller, 요리 = Service, 재료 = Repository"

---

## 1. Spring Boot란 (약 20~30분)

> "예전 Spring은 설정이 길었어요. Boot는 **관례에 맞게 자동 설정**을 많이 해 줘서 **실행 가능한 서버**가 빨리 나와요."

- 내장 Tomcat: "별도 톰캣 설치 없이, JAR 하나로 실행"

---

## 2. Spring Initializr (약 30~40분)

> "브라우저에서 **start.spring.io** 열고, Maven, Java, 의존성만 골라서 ZIP 받으면 됩니다."

- **라이브 데모**: 화면 공유로 Initializr 클릭 → Generate → 압축 해제 → IntelliJ Open
- Group/Artifact 이름 규칙을 한 번만 짚기: "소문자, 패키지 경로에 영향"

### 실행

> "터미널에서 `./mvnw spring-boot:run` — Windows는 `mvnw.cmd`예요."

- IntelliJ에서 `main` 실행하는 방법도 보여 주기.

---

## 3. MVC & 계층 (약 40~50분)

> "요청이 들어오면 **Controller**가 받고, **Service**가 비즈니스 로직, **Repository**가 DB—오늘은 DB 없이 **Controller만** 먼저 해도 됩니다."

- `@RestController`, `@GetMapping`, `@RequestBody` — "REST는 **리소스 + HTTP 메서드**로 생각"

**데모**: `/api/todos` GET 목록, POST 하나 추가

---

## 4. REST 설계 (약 20~30분)

> "URL은 **복수형**이 관례예요. `/api/todos` 처럼요."

- 상태 코드: 200, 201, 404 — "프론트가 에러 처리할 때 구분됨"

---

## 5. Postman (약 15~20분)

> "브라우저는 GET만 쓰기 편해요. **POST/PUT**을 시험하려면 Postman이 편해요."

- New Request → 메서드 선택 → URL → Send

---

## 6. 마무리 (10분)

- 과제: `exercises.md` + Initializr로 만든 프로젝트 GitHub에 올리기
