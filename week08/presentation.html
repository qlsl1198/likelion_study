풀스택 개발 강의

CS/풀스택 퀴즈(단답): React에서 배포용 빌드를 하면 기본으로 생성되는 폴더 이름은?

React + Java Spring Boot

💡 빌드·배포, 한 줄로 말하면?

  • 빌드: 개발자용 코드를 사용자에게 주기에 좋은 형태로 "포장"하는 것
  • 코드 스플리팅: 필요한 것만 먼저 로드 → 첫 화면 빠르게
  • 트리 쉐이킹: 안 쓰는 코드 제거 → 파일 크기 감소
  • 배포: 만든 앱을 인터넷에 올려 누구나 접속 가능하게 함

📋 학습 목표

  • 프로젝트를 빌드하고 최적화할 수 있다
  • 다양한 배포 플랫폼을 이해한다
  • 프로젝트를 배포할 수 있다
  • 코드 리뷰를 할 수 있다
  • 최종 프로젝트를 완성하고 발표할 수 있다

1. 프로젝트 빌드

1.1 React 빌드

npm run build
# build/ 폴더에 생성됨
  • 코드 스플리팅: 번들을 청크로 나누어 초기 로딩 시 필요한 부분만 먼저 로드
  • 트리 쉐이킹: import했지만 사용하지 않는 코드를 제거해 번들 크기 축소

1.2 Spring Boot 빌드

# Maven
./mvnw clean package

# Gradle
./gradlew build

# JAR: target/ 또는 build/libs/

2. 배포 옵션

2.1 프론트엔드 배포

  • Vercel: 가장 간단, 무료
  • Netlify: 정적 사이트 호스팅
  • AWS S3 + CloudFront: 확장성 좋음
  • GitHub Pages: 간단한 프로젝트용

2.2 백엔드 배포

  • AWS EC2: 가상 서버 (직접 관리, 유연함)
  • AWS Elastic Beanstalk: 간편한 배포, 자동 스케일링
  • Railway: 무료 티어 제공, 소규모 프로젝트에 적합
  • Docker + 클라우드: 환경 일관성, 마이크로서비스에 적합

Vercel 배포 - 따라하기 (1/2)

1단계: npm run build 로 빌드 확인

2단계: npm i -g vercel 로 CLI 설치

Mac: 권한 오류 시 sudo npm i -g vercel

3단계: 프로젝트 폴더에서 vercel 실행

Vercel 배포 - 따라하기 (2/2)

4단계: 로그인/가입 후 질문에 응답 (기본값 Enter)

또는: vercel.com 접속 → GitHub 연결 → 저장소 선택 → 자동 배포

Railway 배포 - 따라하기 (1/2)

1단계: railway.app 접속 → GitHub로 로그인

2단계: New Project → Deploy from GitHub → 저장소 선택

3단계: Settings → Build: ./mvnw clean package

Railway 배포 - 따라하기 (2/2)

4단계: Start: java -jar target/todo-api-0.0.1-SNAPSHOT.jar

5단계: Variables에 DB_URL, DB_USERNAME 등 환경 변수 추가
