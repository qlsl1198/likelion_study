"레고 블록처럼 UI를 조각내서 만드는 도구"

1. React 소개 (30분)

1.1 React란?

  • 정의: 사용자 인터페이스를 구축하기 위한 JavaScript 라이브러리
  • 개발사: Facebook (Meta)
  • 특징:
    • 컴포넌트 기반 개발
    • 가상 DOM (Virtual DOM)
    • 단방향 데이터 흐름
    • 재사용 가능한 UI 컴포넌트

가상 DOM (Virtual DOM)이란?

  • 실제 DOM: 브라우저가 화면을 그리기 위해 사용하는 객체 트리. 직접 수정하면 비용이 큼
  • 가상 DOM: React가 메모리에 유지하는 가벼운 복사본
  • 동작 방식: State 변경 → 가상 DOM 비교(diff) → 변경된 부분만 실제 DOM에 반영 → 렌더링 최적화

1.2 React의 장점

  • 컴포넌트 재사용성: 한 번 만든 컴포넌트를 여러 곳에서 사용
  • 가상 DOM: diff 알고리즘으로 최소한의 DOM만 업데이트 → 성능 최적화
  • 풍부한 생태계: 다양한 라이브러리와 도구
  • 커뮤니티: 활발한 개발자 커뮤니티

1.3 React vs Vanilla JavaScript

// Vanilla JavaScript
const button = document.createElement('button');
button.textContent = 'Click me';
button.addEventListener('click', () => {
    alert('Clicked!');
});
document.body.appendChild(button);

// React
function Button() {
    return (
        <button onClick={() => alert('Clicked!')}>
            Click me
        </button>
    );
}

2. JSX 문법 (60분)

2.1 JSX란?

  • 정의: JavaScript XML - JavaScript에 XML/HTML 문법을 추가한 것
  • 특징: React.createElement()의 문법적 설탕
  • 왜 className? JavaScript에서 class는 예약어이므로 className 사용
  • 왜 style은 객체? JSX는 JavaScript이므로 CSS 문자열 대신 객체로 전달

2.2 JSX 기본 문법

// 기본 구조
const element = <h1>Hello, World!</h1>;

// JavaScript 표현식 사용
const name = "John";
const element = <h1>Hello, {name}!</h1>;

// 속성 사용
const element = <img src="image.jpg" alt="Description" />;

// 클래스명 (className 사용)
const element = <div className="container">Content</div>;

2.3 JSX 규칙

1. 하나의 부모 요소

// ❌ 잘못된 예
return (
    <h1>Title</h1>
    <p>Content</p>
);

// ✅ 올바른 예
return (
    <div>
        <h1>Title</h1>
        <p>Content</p>
    </div>
);

JSX 규칙 (계속)

2. Fragment 사용

return (
    <>
        <h1>Title</h1>
        <p>Content</p>
    </>
);

3. 컴포넌트 기초 (90분)

3.1 컴포넌트란?

  • 정의: UI를 독립적이고 재사용 가능한 조각으로 나눈 것
  • 종류:
    • 함수형 컴포넌트 (Function Component) - 권장
    • 클래스형 컴포넌트 (Class Component) - 레거시

3.2 함수형 컴포넌트

// 기본 형태
function Welcome() {
    return <h1>Hello, World!</h1>;
}

// 화살표 함수
const Welcome = () => {
    return <h1>Hello, World!</h1>;
};

// 간단한 경우 중괄호 생략 가능
const Welcome = () => <h1>Hello, World!</h1>;

3.3 컴포넌트 사용

// 컴포넌트 정의
function Welcome() {
    return <h1>Welcome!</h1>;
}

// 컴포넌트 사용
function App() {
    return (
        <div>
            <Welcome />
            <Welcome />
            <Welcome />
        </div>
    );
}

4. Props (60분)

💡 Props vs State, 쉽게 구분하는 법

  • Props: "부모가 건네준 데이터" → 받기만 하고 바꾸지 않음 (예: 제목, 이름)
  • State: "이 컴포넌트가 스스로 갖고 있는 데이터" → 버튼 클릭 등으로 바뀜 (예: 카운트, 체크 여부)
  • 질문: "이 값이 바뀌나?" → Yes: State / No: Props

4.1 Props란?

  • 정의: Properties의 약자
  • 부모 컴포넌트에서 자식 컴포넌트로 데이터를 전달하는 방법
  • 특징: 읽기 전용 (immutable)

4.1 Props vs State

  • Props: 부모에서 전달, 자식이 변경 불가 → 외부 데이터
  • State: 컴포넌트 내부 관리, 변경 시 리렌더링 → 내부 상태

4.2 Props 사용하기

// 부모 컴포넌트
function App() {
    return (
        <div>
            <Greeting name="John" age={25} />
            <Greeting name="Jane" age={30} />
        </div>
    );
}

// 자식 컴포넌트
function Greeting(props) {
    return (
        <div>
            <h1>Hello, {props.name}!</h1>
            <p>You are {props.age} years old.</p>
        </div>
    );
}

4.3 구조 분해 할당

// 방법 1: 함수 매개변수에서 구조 분해
function Greeting({ name, age }) {
    return (
        <div>
            <h1>Hello, {name}!</h1>
            <p>You are {age} years old.</p>
        </div>
    );
}

5. State (90분)

5.1 State란?

  • 정의: 컴포넌트 내부에서 변경 가능한 데이터
  • 특징:
    • State가 변경되면 컴포넌트가 다시 렌더링됨
    • useState Hook을 사용하여 관리

5.2 useState Hook

import { useState } from 'react';

function Counter() {
    // useState 사용법
    // const [상태값, 상태변경함수] = useState(초기값);
    const [count, setCount] = useState(0);
    
    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={() => setCount(count + 1)}>
                Increase
            </button>
        </div>
    );
}

5.3 State 업데이트 - 핵심 개념

함수형 업데이트: setCount(prev => prev + 1)

  • 이전 값 기반으로 갱신 → 비동기·연속 클릭 시 항상 최신 값 반영
  • setCount(count + 1): 현재 count 기준 → 연속 클릭 시 값 꼬일 수 있음
  • setCount(prev => prev + 1): 이전 값 기준 → ✅ 권장

5.3 State 업데이트 - 코드 예시

function Counter() {
    const [count, setCount] = useState(0);
    
    // 직접 증가 (연속 클릭 시 주의)
    const handleIncrease = () => setCount(count + 1);
    
    // 함수형 업데이트 (이전 값 사용) ✅ 권장
    const handleIncrease = () => {
        setCount(prevCount => prevCount + 1);
    };
    
    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={handleIncrease}>Increase</button>
        </div>
    );
}

5.4 여러 State 사용

function Form() {
    const [name, setName] = useState('');
    const [email, setEmail] = useState('');
    const [age, setAge] = useState(0);
    
    return (
        <form>
            <input 
                type="text" 
                value={name}
                onChange={(e) => setName(e.target.value)}
                placeholder="Name"
            />
            <input 
                type="email" 
                value={email}
                onChange={(e) => setEmail(e.target.value)}
                placeholder="Email"
            />
        </form>
    );
}

6. 이벤트 처리 (60분)

6.1 이벤트 핸들러

function Button() {
    const handleClick = () => {
        alert('Button clicked!');
    };
    
    return <button onClick={handleClick}>Click me</button>;
}

6.2 주요 이벤트

  • onClick: 클릭 이벤트
  • onChange: 입력값 변경
  • onSubmit: 폼 제출
  • onFocus: 포커스 받을 때
  • onBlur: 포커스 잃을 때

6.3 폼 처리

function LoginForm() {
    const [email, setEmail] = useState('');
    const [password, setPassword] = useState('');
    
    const handleSubmit = (e) => {
        e.preventDefault();  // 기본 동작 방지
        console.log('Email:', email);
        console.log('Password:', password);
    };
    
    return (
        <form onSubmit={handleSubmit}>
            <input 
                type="email"
                value={email}
                onChange={(e) => setEmail(e.target.value)}
                placeholder="Email"
            />
            <button type="submit">Login</button>
        </form>
    );
}

요구사항

  • 증가/감소 버튼

  • 현재 카운트 표시

예시 코드

import { useState } from 'react';

function Counter() {
    const [count, setCount] = useState(0);
    
    const increase = () => setCount(count + 1);
    const decrease = () => setCount(count - 1);
    const reset = () => setCount(0);
    
    return (
        <div>
            <h1>Counter: <span style={{ color: count < 0 ? 'red' : 'black' }}>
                {count}
            </span></h1>
            <button onClick={increase}>+</button>
            <button onClick={decrease}>-</button>
            <button onClick={reset}>Reset</button>
        </div>
    );
}

2주차 예상 질문 · React 기초

React는 프레임워크인가요, 라이브러리인가요?
공식적으로는 UI를 만드는 데 집중한 자바스크립트 라이브러리로 소개됩니다. 앱 전체를 모두 규정하는 풀 프레임워크라기보다 화면(컴포넌트) 쪽에 가깝게 쓰는 경우가 많아요.
왜 HTML의 class 대신 className을 쓰나요?
JSX는 자바스크립트 안에서 쓰이고, JS에서 class는 예약어라서 충돌을 피하려고 className을 씁니다.
가상 DOM(Virtual DOM)이 뭔가요?
브라우저의 실제 DOM을 매번 통째로 바꾸지 않고, React가 메모리에 둔 가벼운 복사본끼리 비교(diff)해서 바뀐 부분만 반영하려는 방식으로 이해하면 됩니다.
Props와 State는 어떻게 구분하나요?
Props는 부모가 자식에게 넘기는 값(읽기 위주). State는 그 컴포넌트 안에서 바뀌는 데이터입니다. “버튼 눌러서 이 값이 바뀌나?” → 바뀌면 State, 부모가 정한 값이면 Props를 먼저 생각해 보세요.
useState가 [값, 함수] 두 개를 주는 이유는요?
첫 번째는 현재 state, 두 번째는 setter입니다. state에 직접 대입하면 React가 변경을 제대로 반영하지 못할 수 있어요.
setCount(count + 1)과 setCount(prev => prev + 1) 차이는요?
연속으로 여러 번 업데이트할 때는 함수형 업데이트(prev => prev + 1)이 안전합니다.
return 안에 왜 하나의 부모로 감싸야 하나요?
JSX는 하나의 루트를 반환해야 합니다. 불필요한 div가 싫으면 Fragment(<>...</>)를 쓰세요.
컴포넌트 이름은 왜 대문자로 시작하나요?
소문자면 HTML 태그로 해석될 수 있어요. React 컴포넌트는 대문자 시작이 규칙입니다.
onClick에 왜 문자열이 아니라 함수를 넣나요?
React에서는 중괄호 안에 자바스크립트 함수를 넘깁니다.
폼에서 e.preventDefault()는 왜 쓰나요?
기본 제출(새로고침 등)을 막고 자바스크립트로만 처리하려고 씁니다.
create-react-app 말고 Vite 등은요?
실무에서는 Vite 등도 많이 씁니다. 강의는 CRA로 안내했지만 개념은 같고 도구만 다르다고 이해하면 됩니다.
