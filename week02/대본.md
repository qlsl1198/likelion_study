# 2주차 강의 대본 — React 기초

**예상 시간**: 약 3시간  
**준비물**: 1주차까지 Node·VS Code 설치 완료, React 프로젝트 생성 가능한 상태

---

## 오프닝 (5~10분)

> "지난주에 환경 잡았죠. 오늘부터는 **화면을 만드는 쪽**에 집중합니다. **React**는 페이스북이 만든 UI 라이브러리고, 지금은 프론트엔드 채용 공고에 거의 다 나올 정도로 표준이에요.  
> 오늘 목표는 **JSX로 화면을 쓰고**, **컴포넌트로 쪼개고**, **Props와 State로 데이터를 흐르게** 하는 것까지예요."

- "레고 블록" 비유로 컴포넌트 한 번 더 짚기.

---

## 1. React 소개 (약 30분)

### 왜 React인가

> "바닐라 JS로도 버튼 만들 수 있죠. 근데 화면이 커지면 **코드가 꼬여요**. React는 **화면을 함수/조각으로 나눠서** 생각하게 해 줍니다."

- 가상 DOM: "비유로는 **바뀐 부분만 다시 그리는 스마트한 방식**" 정도만. 세부 알고리즘은 선택.
- **멘트**: "처음엔 문법이 낯설어요. **JSX = HTML 비슷한 것처럼 보이는 JS**라고만 기억하세요."

### 슬라이드에 있는 Vanilla vs React 예제

> "왼쪽은 DOM을 직접 만지는 거고, 오른쪽은 **return 안에 UI를 그린다**는 느낌이에요."

---

## 2. JSX (약 60분)

> "JSX에서는 **className**이에요, class가 아니에요. JavaScript에서 class는 예약어라서요."

- 중괄호 `{ }` 안에 변수·식 넣기: "중괄호가 **자바스크립트 구역**이에요"
- Fragment `<>...</>`: "불필요한 div로 감싸기 싫을 때"

**데모**: 작은 컴포넌트 하나를 라이브에서 수정해 저장 → 화면 반영

---

## 3. 컴포넌트 (약 30~40분)

> "함수 이름을 **대문자로** 시작하면 React가 컴포넌트로 인식해요."

- `Welcome`을 여러 번 쓰는 예제: "재사용이 바로 보이죠"

---

## 4. Props (약 40~50분)

> "Props는 **부모가 자식에게 넘겨주는 값**이에요. 읽기만 해요, 자식이 마음대로 바꾸지 않아요."

- Props vs State: "바뀌는 건 주로 State, **부모가 정해준 건** Props"

**멘트**: "헷갈리면 **'이 값이 버튼 눌러서 바뀌나?'** → 바뀌면 State, 아니면 Props"

---

## 5. State & useState (약 50~60분)

> "State가 바뀌면 React가 **그 컴포넌트를 다시 그려요**. 그게 리렌더링이에요."

- `useState`: `[값, setter]` — "setter로만 바꾸세요, 직접 대입하면 안 돼요"
- 함수형 업데이트 `setCount(c => c+1)`: "연속 클릭할 때 안전하다" 정도만

**데모**: 카운터 버튼 하나

---

## 6. 이벤트 (약 20~30분)

> "DOM에서는 `onClick`이 문자열이었죠. React는 **중괄호 안에 함수**를 넣어요."
