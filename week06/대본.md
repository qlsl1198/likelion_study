# 6주차 강의 대본 — 프론트엔드·백엔드 연동

**예상 시간**: 약 3시간  
**준비물**: React 앱(localhost:3000), Spring Boot 앱(localhost:8080), 두 터미널

---

## 오프닝 (5~10분)

> "지금까지는 브라우저 따로, 서버 따로였죠. 오늘은 **붙입니다**.  
> 여기서 중요한 게 **CORS**예요—**다른 포트**는 브라우저가 다른 출처로 봐서, 서버가 **허용해줘야** 해요."

- Axios: "fetch보다 설정·에러 처리가 편한 경우가 많아요"

- 타이틀 슬라이드 퀴즈 정답(강사용): `Same-Origin Policy`

---

## 1. CORS (약 30~40분)

> "Same-Origin Policy—**프로토콜, 도메인, 포트**가 같아야 같은 출처예요. 그래서 3000과 8080은 **다른 출처**입니다."

- Spring에서 `allowedOrigins`에 `http://localhost:3000` 넣기
- **멘트**: "배포할 때는 프론트 도메인으로 바꿔야 해요"

**데모**: CORS 설정 전/후로 브라우저 콘솔 에러 비교

---

## 2. Axios (약 40~50분)

> "`npm install axios` 하고, `axios.get`, `axios.post`로 호출해요."

- 인스턴스: `baseURL`을 한 번에 넣으면 편함
- `then`/`catch` 또는 `async/await` — "팀 컨벤션에 맞추면 됩니다"

**데모**: Todo 목록 GET → 화면에 리스트

---

## 3. 로딩·에러 (약 30분)

> "API는 항상 **늦게** 올 수 있어요. **로딩 중**, **에러** 상태를 state로 두는 패턴을 보여 주세요."

---

## 4. 동시 실행 (10분)

> "터미널 하나는 백엔드, 하나는 `npm start`—**둘 다 켜져 있어야** 연동됩니다."

---

## 5. 실습·마무리 (20분)

> "Todo CRUD를 한 화면에서 다 붙여 보는 게 과제 목표예요. 막히면 **Network 탭**과 **CORS 에러 메시지**를 먼저 보라고 안내하세요."

---

## 마무리 멘트

> "다음 주는 **로그인**, **JWT**, **토큰**입니다. 오늘은 **누구나 API를 쓸 수 있는 상태**였고, 다음엔 **막을 거예요**."
