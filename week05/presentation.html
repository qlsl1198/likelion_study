💡 JPA, 한 줄로 말하면?

"Java 객체 ↔ DB 테이블을 자동으로 연결해 주는 것"

  • JDBC: SQL을 직접 짜서 DB 접근 (번거로움)
  • JPA: Java 코드만 쓰면 SQL은 알아서 생성
  • 예: todoRepository.save(todo) → INSERT 쿼리 자동 실행

📋 학습 목표

  • JPA/Hibernate를 이해하고 사용할 수 있다
  • Spring Data JPA를 활용할 수 있다
  • 엔티티 관계를 설계하고 매핑할 수 있다
  • 데이터베이스 연동을 구현할 수 있다
  • 쿼리 메서드를 작성할 수 있다

1. JPA/Hibernate 소개

1.1 JPA란?

  • 정의: Java 객체와 DB 간 매핑을 위한 표준
  • JPA vs JDBC: JDBC는 SQL 직접 작성·반복 코드多. JPA는 객체 중심, 자동 SQL 생성, 유지보수 용이
  • 장점: 객체지향적 접근, DB 독립성, 생산성 향상

2.1 @Entity 어노테이션

@Entity
@Table(name = "todos")
public class Todo {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(nullable = false, length = 100)
    private String title;
    
    private boolean completed;
    
    @CreatedDate
    private LocalDateTime createdAt;
    
    @LastModifiedDate
    private LocalDateTime updatedAt;
}

3.1 Repository 인터페이스

public interface TodoRepository extends JpaRepository<Todo, Long> {
    // save, findById, findAll, deleteById 등 기본 제공
    List<Todo> findByCompleted(boolean completed);
}

3.3 @Query 어노테이션

@Query("SELECT t FROM Todo t WHERE t.completed = :completed")
List<Todo> findTodosByCompleted(@Param("completed") boolean completed);

@Query(value = "SELECT * FROM todos WHERE completed = :c", nativeQuery = true)
List<Todo> findNative(@Param("c") boolean completed);

4.1 @OneToMany / @ManyToOne

// User
@OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
private List<Todo> todos;

// Todo
@ManyToOne(fetch = FetchType.LAZY)
@JoinColumn(name = "user_id")
private User user;

4.2 FetchType

  • LAZY: 필요할 때만 로딩 → N+1 주의하되 성능에 유리
  • EAGER: 즉시 로딩 → 불필요한 조인 발생 가능

5.1 application.properties 설정 - 따라하기

src/main/resources/application.properties 에 아래 내용 추가:

# application.properties
spring.datasource.url=jdbc:postgresql://localhost:5432/tododb
spring.datasource.username=postgres
spring.datasource.password=yourpassword
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

5.2 ddl-auto 옵션 (1/2)

  • create: 시작 시 테이블 생성, 종료 시 삭제 (초기화용)
  • update: 스키마 업데이트 (개발용 권장)
  • validate: 스키마 검증만 (운영용 권장)
  • none: 아무 작업 안 함

5.2 ddl-auto 옵션 (2/2)
